# OSPF Virtual Link

In [[Open Shortest Path First (OSPF)|OSPF]], a **virtual link** is a logical tunnel that connects a router (or area) that is not physically adjacent to [[OSPF Area 0 (Backbone)|Area 0]] to the backbone.

## Purpose

- Allows an area to connect to the backbone through another non-backbone area
- Configured when a router cannot be directly connected to Area 0
- Exchanges backbone routing information over the tunnel
