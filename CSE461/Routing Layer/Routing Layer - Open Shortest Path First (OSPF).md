# Routing Layer: Open Shortest Path First (OSPF)

## Low-Level Primer: The Link-State Standard
**[[OSPF]]** is a hierarchical, link-state intradomain routing protocol. It is the most widely deployed Interior Gateway Protocol (IGP) in large enterprise networks. OSPF routers discover neighbors using a **Hello Protocol** and exchange **Link-State Advertisements (LSAs)** to build a synchronized **Link-State Database (LSDB)**—a complete "map" of the network.

---

## 1. Hierarchical Routing: OSPF Areas

To scale to thousands of routers, OSPF divides an **[[Autonomous System (AS)]]** into **Areas**. This limits the scope of LSA flooding and reduces the CPU overhead of Dijkstra calculations.

### Area Types
*   **Backbone Area (Area 0)**: The core of the OSPF network. All other areas **must** connect to Area 0. All inter-area traffic passes through the backbone.
*   **Non-Backbone Areas**: Standard areas where local routing occurs.

### Router Roles
*   **Internal Router (IR)**: All interfaces belong to a single area.
*   **Area Border Router (ABR)**: Connects one or more areas to the backbone. It summarizes routing information (aggregates prefixes) before passing it to other areas.
*   **Backbone Router (BR)**: Any router with at least one interface in Area 0.
*   **AS Boundary Router (ASBR)**: A router that exchanges routing information with other Autonomous Systems (e.g., via BGP).

## 2. LSA Types and Scope

OSPF uses different packets to describe different parts of the topology:

| Type | Name | Scope | Description |
| :--- | :--- | :--- | :--- |
| **1** | **Router LSA** | Intra-area | Describes a router's local links and their costs. |
| **2** | **Network LSA** | Intra-area | Generated by the **Designated Router (DR)** for multi-access segments (e.g., Ethernet). |
| **3** | **Summary LSA** | Inter-area | Generated by **ABRs** to describe routes to other areas. |
| **5** | **AS-External** | AS-wide | Describes routes to destinations outside the OSPF AS. |

---

## 3. Multi-Access Segments: DR and BDR

On shared media like Ethernet, $N$ routers would form $N(N-1)/2$ adjacencies, creating an $O(N^2)$ flooding problem.

*   **Designated Router (DR)**: A single router elected to be the central point for LSA exchange on the segment. All other routers form an adjacency **only** with the DR and the BDR.
*   **Backup Designated Router (BDR)**: Monitors the DR and takes over if it fails.
*   **Multicast Addresses**:
    *   **224.0.0.5 (AllSPFRouters)**: Used by the DR to flood updates to all routers.
    *   **224.0.0.6 (AllDRouters)**: Used by routers to send updates to the DR/BDR.

---

## 4. OSPF Packet Types (Layer 3)

OSPF runs directly over IP using **Protocol 89**.
1.  **Hello**: Neighbor discovery and keepalives.
2.  **Database Description (DBD)**: Summarizes the LSDB during initial synchronization.
3.  **Link-State Request (LSR)**: Requests missing LSAs from a neighbor.
4.  **Link-State Update (LSU)**: Carries the actual LSAs.
5.  **Link-State ACK**: Acknowledges LSUs to ensure reliable flooding.
