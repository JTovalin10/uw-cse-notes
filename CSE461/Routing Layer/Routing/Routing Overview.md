# Intradomain Routing: Distance-Vector vs. Link-State

**Intradomain Routing** (or Interior Gateway Protocol, **IGP**) is the process by which routers within a single [[Autonomous System (AS)]] exchange reachability information to build their **Forwarding Tables**. The goal is to determine the shortest path to every node based on a specific **Metric** (e.g., hop count, bandwidth, or delay).

---

## 1. Distance-Vector Routing

Distance-Vector (DV) is based on the **Bellman-Ford** algorithm. Each router maintains a table of distances to all known destinations and periodically shares this table with its **immediate neighbors**.

### Low-Level Mechanics
*   **The DV Equation**: Each router $x$ maintains a vector $D_x = [D_x(y): y \in V]$. When $x$ receives a vector from neighbor $v$:
    $$D_x(y) = \min_{v \in neighbors}\{cost(x, v) + D_v(y)\}$$
*   **Local Knowledge**: Routers only know the cost to their neighbors and the advertised distances from those neighbors. They do not know the full network topology.
*   **Convergence**: DV is "iterative" and "distributed." Convergence can be slow during topology changes.

### The "Count-to-Infinity" Problem
Occurs when a link fails, and two routers keep incrementing their distances to a destination based on each other's stale information.
*   **Solutions**:
    *   **Split Horizon**: A router does not advertise a route back to the neighbor from which it learned that route.
    *   **Poison Reverse**: A router advertises a distance of $\infty$ for a route back to the neighbor it learned it from.
    *   **Triggered Updates**: Send updates immediately when a topology change is detected rather than waiting for the next periodic timer.

### Protocol Example: RIP (Routing Information Protocol)
*   **Metric**: Simple **Hop Count** (Max = 15; 16 is considered $\infty$).
*   **Transport**: Runs over **[[User Datagram Protocol (UDP)]]** on port **520**.
*   **Update Interval**: 30 seconds.

## 2. Link-State Routing

Link-State (LS) is based on **Dijkstra’s Algorithm**. Each router discovers its neighbors, builds a **Link-State Packet (LSP)**, and floods it to **every router** in the network.

### Low-Level Mechanics
*   **Global Knowledge**: Every router maintains an identical **Link-State Database (LSDB)** representing the complete network map.
*   **Reliable Flooding**: When a link state changes, the updated LSP is flooded using sequence numbers and ACKs to ensure every node receives the latest version.
*   **Shortest Path First (SPF)**: Each router independently runs Dijkstra’s algorithm on its local LSDB to compute its own shortest-path tree.
    $$dist(v) = \min(dist(v), dist(u) + cost(u, v))$$

### Advantages of Link-State
*   **Fast Convergence**: Since every node has the full map, they can react immediately to changes once the LSP arrives.
*   **Robustness**: An incorrect LS advertisement only affects the routes passing through that specific node, whereas an incorrect DV update can propagate through the entire network.

### Protocol Example: OSPF (Open Shortest Path First)
*   **Metric**: Configurable **Cost** (usually inversely proportional to bandwidth).
*   **Transport**: Runs directly over **[[Internet Protocol (IP)]]** (Protocol 89).

---

## Comparison Summary

| Feature | Distance-Vector (DV) | Link-State (LS) |
| :--- | :--- | :--- |
| **Algorithm** | Bellman-Ford | Dijkstra (SPF) |
| **Knowledge** | Neighbors only (Vectors) | Global (Full Topology Map) |
| **Convergence** | Slow (potential loops) | Fast |
| **Scalability** | Good (low memory) | Medium (CPU/Memory intensive) |
| **Messaging** | Periodic updates to neighbors | Flooding to all nodes |

## Related Topics

*   **[[Forwarding Table]]**: The final output generated by these algorithms.
*   **[[BGP (Border Gateway Protocol)]]**: Used for routing *between* Autonomous Systems.
*   **[[Spanning Tree Algorithm]]**: The Layer 2 equivalent for preventing loops in switched networks.
