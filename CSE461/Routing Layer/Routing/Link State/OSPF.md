# Open Shortest Path First (OSPF)

**OSPF** is a hierarchical, **Link-State** intradomain routing protocol. It is the most widely deployed Interior Gateway Protocol (IGP) in large enterprise and provider networks due to its fast convergence and support for complex topologies.

## Low-Level Primer: The Link-State Database (LSDB)
*   **Adjacency**: OSPF routers first discover neighbors via the **Hello Protocol**. If they share a common link and compatible parameters (timers, MTU), they form an **Adjacency**.
*   **Link-State Advertisement (LSA)**: Each router generates LSAs describing its local links and their costs.
*   **Reliable Flooding**: LSAs are flooded to all routers in the OSPF domain. Every router maintains an identical **LSDB**, which is a synchronized "map" of the entire network.
*   **SPF Calculation**: Each router runs **Dijkstra's Algorithm** on its LSDB to find the shortest path to all destinations, which is then used to populate the [[Forwarding Table]].

## Hierarchical Routing: OSPF Areas
To scale OSPF to very large networks, a single Autonomous System is divided into **Areas**. This limits the scope of LSA flooding and reduces the size of the LSDB on individual routers.

### Area Types
*   **Backbone Area (Area 0)**: The core of the OSPF network. All other areas must connect to Area 0.
*   **Non-Backbone Areas**: Standard areas where local routing occurs. Traffic between two non-backbone areas **must** pass through the backbone.

### Router Roles
*   **Internal Router (IR)**: All interfaces belong to a single area.
*   **Area Border Router (ABR)**: Connects one or more areas to the backbone. It summarizes routing information between areas.
*   **Backbone Router (BR)**: Any router with at least one interface in Area 0.
*   **AS Boundary Router (ASBR)**: A router that exchanges routing information with other Autonomous Systems (e.g., via **BGP**).

## LSA Types and Propagation
OSPF uses different types of LSAs to describe different parts of the topology:

| LSA Type | Name | Scope | Purpose |
| :--- | :--- | :--- | :--- |
| **Type 1** | **Router LSA** | Intra-Area | Describes a router's local links and costs within its area. |
| **Type 2** | **Network LSA** | Intra-Area | Generated by the **Designated Router (DR)** to describe multi-access segments (e.g., Ethernet). |
| **Type 3** | **Summary LSA** | Inter-Area | Generated by **ABRs** to describe routes to other areas. |
| **Type 4** | **ASBR Summary** | Inter-Area | Informs the AS how to reach an **ASBR**. |
| **Type 5** | **AS External** | AS-Wide | Describes routes to destinations outside the OSPF AS. |

## Mechanics of Multi-Access Links (DR/BDR)
On shared networks like Ethernet, $N$ routers would form $N(N-1)/2$ adjacencies, leading to excessive flooding.
*   **Designated Router (DR)**: A single router elected to be the focal point for LSA exchange on the segment.
*   **Backup Designated Router (BDR)**: Assumes the DR role if the DR fails.
*   **AllDRouters (224.0.0.6)**: Multicast address used to send updates to the DR/BDR.
*   **AllSPFRouters (224.0.0.5)**: Multicast address used by the DR to flood updates to all other routers.

## OSPF Packet Types
OSPF runs directly over **IP (Protocol 89)**.

1.  **Hello**: Discovers and maintains neighbor adjacencies.
2.  **Database Description (DBD)**: Used during synchronization to describe LSDB contents.
3.  **Link-State Request (LSR)**: Requests missing LSAs from a neighbor.
4.  **Link-State Update (LSU)**: Carries the actual LSAs (response to LSR or triggered update).
5.  **Link-State ACK**: Acknowledges receipt of an LSU.
