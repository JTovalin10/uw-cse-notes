# CSE391: Introduction to Linux

An **[[System and Software Tools/Linux Fundamentals/Operating System|Operating System]]** is the software that manages software and hardware resources, such as computer memory, network connections, peripherals, and users.

## Unix and Linux

**Unix** was a proprietary operating system developed at Bell Labs in the late 1960s and early 1970s by figures such as Dennis Ritchie and Ken Thompson. **Linux** is a group of free and open-source Unix-like operating systems, started in 1991 by Linus Torvalds.

### Key Ideas Introduced by Unix/Linux
- Combining many small programs to solve one large problem.
- Multiple users with different permissions.
- Hierarchical file system with recursive folders.
- **"Everything is a file"** philosophy.
- Documentation included with the software.

### Impact
- Powering most web servers and cloud computing.
- Foundations for macOS, iOS, Android, and various Linux distributions (Ubuntu, Debian, Fedora, etc.).

## The Shell

A **[[Shell|Shell]]** is an interactive program that allows the user to interact with the operating system and its applications. It is almost always text-based, as opposed to a **Graphical User Interface (GUI)**.

### Why Use a Shell?
- Efficiency and speed for complicated tasks.
- Working on remote computers (e.g., via **SSH**).
- High programmability and customizability.

### Accessing Linux
Common methods to access a Linux environment include:
- Remotely logging in via `ssh` to servers like `attu` (CSE majors) or `vergil` (all UW students).
- Running a **Virtual Machine (VM)**.
- Using the **Windows Subsystem for Linux (WSL)**.
