Standard Streams
Unix processes have three streams: abstract locations that tell a program
where to read input from and where to write output to.
There are three standard streams:
●
●
●

stdin (standard input)
stdout (standard output)
stderr (standard error)

You’ve implicitly used these when writing Java programs—the System.out in
System.out.println is referring to stdout!

Standard Input versus Parameters
One of the most important distinctions in this class is the diﬀerence
between standard input (stdin) and a command’s parameters:
●

parameters are arguments given on the command line:
○
○

●

e.g. in ls dir1,
e.g. in grep "a",

dir1 is a parameter
"a" is a parameter

standard input comes from the user (either from a ﬁle or the console)
○
○

e.g. in grep "a" < berries.txt, the contents of berries.txt go into stdin
e.g. after typing grep "a", it accepts input from your keyboard until the stream is closed
(using Ctrl + D)
