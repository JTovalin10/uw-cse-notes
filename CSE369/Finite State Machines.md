# Course: Finite State Machines

A **Finite State Machine** (FSM) is a mathematical model of computation used to design sequential logic circuits. Unlike combinational logic, FSMs have "memory" provided by state elements (flip-flops).

## Components of an FSM
1. **Inputs**: External signals that influence transitions.
2. **Outputs**: Signals generated by the machine.
3. **States**: The internal "memory" of the machine.
4. **State Transitions**: Logic that determines the next state based on current state and inputs.
5. **Output Logic**: Logic that determines outputs based on state (and sometimes inputs).

## FSM Types
- **Moore Machine**: Outputs depend *only* on the current state.
- **Mealy Machine**: Outputs depend on *both* the current state and the current inputs. Mealy machines can often represent the same logic with fewer states but may be more susceptible to glitches.

## Implementation Steps
1. **State Diagram**: A visual representation of states (circles) and transitions (arrows).
2. **State Table**: A tabular version of the state diagram.
3. **State Encoding**: Assigning binary values to each state (e.g., Binary, One-hot, Gray code).
4. **Logic Minimization**: Using K-maps to find the minimal logic for next-state and output functions.
5. **Circuit Realization**: Implementing with flip-flops and logic gates.

### Formal Definition
A quintuple $(Σ, S, s_0, δ, ω)$, where $Σ$ is the input alphabet, $S$ is the set of states, $s_0$ is the initial state, $δ$ is the state-transition function, and $ω$ is the output function.

### Simplified Explanation
A machine that remembers what it was doing. It stays in a "state" until an input tells it to move to a different state, like a vending machine or a traffic light.
