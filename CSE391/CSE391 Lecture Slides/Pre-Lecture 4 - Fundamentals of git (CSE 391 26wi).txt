Branches as pointers (or references)
A branch in git is a pointer/reference to a speciﬁc commit.
HEAD: the current branch (~ commit) you are working on.
main: typically the “default” branch or source of truth

Why branches?
So far, we’ve done all our work on the main branch.
However, you typically work on a separate branch:
●

main is the single source of truth – it should be stable (e.g. compile)
○
○

●

can’t share or collaborate on in-progress work, since that would break the rule
also hard to work on two (or more) unrelated features at once

instead, you’ll do your work on a separate branch and “merge” it in
○
○

we’ll ease you in to this process in HW4 & HW5!
for this video, we’ll slightly break this rule – mostly for demonstration purposes

Note: main used to be called “master”, though conventions have changed recently. You may see both
