# CSE391: Remote Repositories

A **Remote** repository is a version of your project that is hosted on the Internet or network somewhere (like **GitHub** or **[[GitLab]]**).

## Syncing with Remotes
- **[[clone]]**: Create a local copy of a remote repository.
- **fetch**: Download changes from the remote but do not merge them into your local work.
- **pull**: Fetch and then merge changes from the remote into your current branch.
- **push**: Upload your local commits to the remote.

## Origin
`origin` is the default name Git gives to the server you cloned from. `origin/main` refers to the state of the `main` branch on that remote server.
