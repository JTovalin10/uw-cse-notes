# Memory Consistency Model

**[[Memory Consistency Model]]**: (MCM) A contract between the hardware/system and the software programmer that specifies the rules for how memory operations (reads and writes) appear to different processors.

## Purpose
It defines which reorderings by the compiler or hardware are legal, allowing the programmer to reason about the correctness of concurrent code.
