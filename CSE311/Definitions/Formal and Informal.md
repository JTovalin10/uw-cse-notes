# CSE 311: Formal and Informal

A **formal** description is built on a precise set of rules for **syntax** (how expressions are written) and **semantics** (what they mean), so a statement can only be interpreted one way. An **informal** description uses natural language to explain what a formal system does, prioritizing human intuition over machine-level precision and so possibly carrying ambiguity.
