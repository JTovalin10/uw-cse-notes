
A **formal** system is built on a precise set of rules for its syntax (how expressions are written) and semantics (what they mean). The goal is to eliminate all ambiguity so that a statement or expression can only be interpreted in one way. This is crucial for computers and mathematical proofs. ü§ñ

**Key Characteristics:**

- **Unambiguous**: Each statement has exactly one meaning.
    
- **Rule-Based**: Follows a strict grammar and logic.
    
- **Symbolic**: Often uses specialized symbols and notation.
    

**Examples from our conversation:**

- The type definition `type Beat := {count : N, up : B}` is **formal**.
    
- The function pattern `dance({count: 0, up: T}) := 1` is **formal**. It's a piece of code that a machine or another computer scientist can parse and understand precisely.
